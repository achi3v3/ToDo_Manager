1. Сервис задач (To-Do API) с REST API и базой данных

Описание:
    Простое приложение для управления задачами (CRUD), но с расширенными возможностями.

Что ты освоишь:

    Горутины и каналы:
        Асинхронная обработка задач (например, отправка уведомлений о дедлайнах в фоне).
        Логирование операций через отдельную горутину.

    REST API:
        Эндпоинты для создания, чтения, обновления и удаления задач (GET /tasks, POST /tasks, PUT /tasks/{id}, DELETE /tasks/{id}).
        Пагинация, фильтрация (например, GET /tasks?status=completed).

    SQL и БД:
        Работа с PostgreSQL (или SQLite для простоты).
        Схема: tasks (id, title, description, status, deadline, created_at).
        Использование database/sql или ORM (например, GORM).

    Error handling:
        Кастомные ошибки (например, TaskNotFound, InvalidDeadline).
        Middleware для обработки ошибок в HTTP.

    Фреймворки:
        Можно использовать Gin или Echo для роутинга.

    Дополнительно: подключить Swagger для документации API.

    Расширения (если захочешь усложнить):
        Добавить аутентификацию (JWT).
        Кэширование с Redis.
        Тесты (unit, интеграционные).
